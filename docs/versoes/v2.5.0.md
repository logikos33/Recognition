# v2.5.0 — 2026-04-19

## Tipo de Incremento
MINOR — nova feature completa sem quebra de compatibilidade com APIs existentes

## Escopo
Quality Gate RVB Isolantes — rastreamento de peças em fluxo multi-bancada com inspeção YOLO em 3 validações

## Resumo Executivo

Implementação do fluxo de Quality Gate para o cliente RVB Isolantes. O sistema rastreia cada peça desde a entrada na Bancada A até a aprovação final na Bancada B, passando por até 3 ciclos de validação com câmera YOLO e retrabalho rastreado. A peça aprovada é exportada automaticamente para o sistema Wiser.

## Componentes Entregues

### Banco de Dados
- Migration 033 idempotente aplicada em todos os schemas de tenant via `public.tenants`
- `quality_pieces` — state machine de peças (11 estados, 2 terminais)
- `quality_reworks` — histórico de retrabalho com fotos antes/depois e duração
- `quality_wiser_exports` — log de exportações com retry automático
- `quality_stations` — configuração de bancadas (câmeras, torre, tablet URL)
- `quality_inspections` recebe 8 novas colunas RVB
- `quality_camera_config` recebe 6 novas colunas RVB
- `create_tenant_schema()` atualizada — novos tenants recebem as tabelas automaticamente

### Backend
- `PieceStateMachine` — 11 estados, transições validadas, estados terminais bloqueados
- `GateRepository` — CRUD com psycopg2 + RealDictCursor, filtros de listagem, overview_stats, rework_stats
- `GateService` — orquestra state machine, Redis publish, Celery dispatch, tower signal, Wiser export
- `OcrService` — PaddleOCR com fallback pyzbar para leitura do número da peça
- `TowerController` — ABC adapter com 3 implementações; seleção por `TOWER_CONTROLLER_TYPE` env var
- `PhotoService` — save local + upload R2; fallback silencioso em caso de falha R2
- `WiserIntegration` — 3-mode fallback: API REST > file share > PDF
- 17 endpoints REST em `/api/v1/quality/gate/*`
- 4 canais Redis no namespace `/quality` (per-tenant por schema)
- Celery task `run_quality_gate_inspection` (N-frame voting)
- Celery Beat `quality-wiser-retry` (5 min)

### Frontend
- Rota pública `/tablet/:station` — TabletKiosk com 7 telas, sem JWT
- `useTabletWebSocket` — hook WebSocket filtrado por station code
- 5 páginas de dashboard: Gate, Pecas, Retrabalho, Relatorios, Config
- QualityLayout com 5 novos itens de navegação

### Testes
- 71 testes passando, 1 skipped (reportlab ausente no env de teste), 0 falhas
- Cobertura: state_machine, gate_service, gate_repository, ocr_service, tower_controller, wiser_integration, photo_service, gate_routes

## Variáveis de Ambiente Novas

| Variável | Default | Descrição |
|----------|---------|-----------|
| `TOWER_CONTROLLER_TYPE` | `simulated` | Implementação da torre: `simulated`, `gpio`, `http_relay` |
| `TOWER_HTTP_RELAY_URL` | — | URL do relay HTTP (obrigatório se `http_relay`) |
| `WISER_API_URL` | — | URL da API Wiser REST (opcional) |
| `WISER_FILE_SHARE_PATH` | — | Caminho da pasta compartilhada Wiser (opcional) |
| `WISER_PDF_OUTPUT_PATH` | `./wiser_exports` | Diretório de saída para PDFs Wiser |

## Dívidas Conhecidas Desta Versão

1. `success()` helper não aceita `status_code` — endpoint de criação de station retorna 200 em vez de 201
2. `R2Storage.get_instance()` não implementado — uploads de foto caem silenciosamente sem R2 key
3. `reportlab` não instalado no env de testes — 1 teste de wiser_integration skipped

## Compatibilidade

- Banco: migration idempotente, não há `DROP`; retrocompatível com tenants existentes
- API: novos endpoints em `/gate/*` não afetam endpoints existentes do módulo quality
- Frontend: novas rotas adicionadas sem remover existentes; rota `/tablet/:station` é pública (sem impacto em auth)
