# v2.6.0 — 2026-05-03

## Tipo de Incremento
MINOR — nova feature completa (white-label UI + design system) sem quebra de compatibilidade com APIs existentes

## Escopo
Recognition Rebrand — Sprints 6 e 7: Admin White-Label UI e polimento final

## Resumo Executivo

Conclusão do rebrand Recognition. Sprint 6 entregou o fluxo de configuração de identidade visual por tenant no painel admin, com editor de cores (verificação WCAG AA inline), preview ao vivo em 3 telas (Login, Dashboard, Andon) e sandbox de paletas. Sprint 7 eliminou as últimas referências ao nome anterior "EPI Monitor" e criou a página DesignSystem para documentação viva de tokens e componentes primitivos.

## Componentes Entregues

### Sprint 6 — Admin White-Label UI

#### Componentes Reutilizáveis
- `ColorPicker` — input de cor com verificação de contraste WCAG AA; calcula luminância relativa e exibe ratio (ex: `4.52:1 AA ✓` / `2.31:1 AA ✗`); parâmetro `bgBase` configurável para comparação
- `BrandingPreview` — 3 mini-telas em CSS inline refletindo `primary`, `accent` e `productName` em tempo real: Login (logo, formulário, botão), Dashboard (top bar, KPI cards, gráfico de barras), Andon (grid 2x2 de câmeras, métricas de conformidade/alertas)
- `TenantBrandingEditor` — formulário controlado: nome do produto, cor primária, cor de acento, upload de logo (FileReader → data URL); botão de remoção de logo

#### Páginas Admin
- `AdminBrandingTenantsPage` (`/admin/branding/tenants`) — listagem de tenants com swatches duplos (primary + accent), indicador "Customizado" quando override existe em localStorage, navegação para editor
- `AdminBrandingEditorPage` (`/admin/branding/tenants/:id`) — layout 2 colunas (editor + preview sticky); salva/carrega de `localStorage`; modo "Visualizar como tenant" injeta CSS vars via `resolveTheme()` no `<style id="recognition-tenant-theme">`; botão "Resetar padrão" remove key do localStorage
- `AdminBrandingDefaultPage` (`/admin/branding/default`) — referência somente-leitura; 6 grupos, 22 tokens (fundo, texto, primary, accent, semânticas, bordas); cada entrada tem swatch, nome, valor e descrição
- `AdminBrandingSandboxPage` (`/admin/branding/sandbox`) — 6 presets de paleta; `ColorPicker` livre; botão "Aplicar na página" para teste in-app sem persistência

#### Infraestrutura Admin
- `AdminLayout.tsx` atualizado: grupo "Identidade Visual" na sidebar com item "White-label"; 4 rotas `/admin/branding/*` registradas; ícone `Palette` importado de `lucide-react`

### Sprint 7 — Polimento e Auditoria

#### Remoção de Referências ao Nome Anterior
- `types/index.ts` — referências "EPI Monitor" substituídas por "Recognition"
- `ModuleSelectionPage` — texto do módulo atualizado para "Recognition"
- `TabletIdle` — label do produto atualizado para usar `productName` do tema
- Todos os hexs hardcoded intencionais receberam comentário `// allow:` para silenciar lint com rastreabilidade

#### Design System
- `DesignSystemPage` (`/design-system`) — catálogo interno superadmin-only com seções:
  - Color Tokens (20 tokens com swatch, nome, valor)
  - Typography (8 escalas: Display, H1, H2, H3, Body, Small, Caption, Label)
  - Spacing (6 tokens: xs=4px a xxl=48px)
  - Button (4 variantes: primary/secondary/ghost/danger; 3 tamanhos: sm/md/lg)
  - Badge (6 variantes: success/warning/danger/primary/neutral/accent)
  - Input (4 estados: padrão, com label, com erro, desabilitado)
  - Skeleton (4 variantes: title/text/rect + composição)
  - Toast (4 tipos: success/error/warning/info interativos)
  - Shadow (5 tokens: sm/md/lg/glow/glowCyan)
  - Border Radius (5 tokens: sm/md/lg/xl/full)
- `AppRoutes.tsx` atualizado: rota `/design-system` lazy-loaded dentro de `AdminRoute`, fora do `AdminLayout` para visualização limpa

3. `/design-system` sem busca ou âncoras de seção (aceitável para catálogo interno)
